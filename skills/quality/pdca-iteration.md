# PDCA 자동 개선

> PDCA 이터레이터가 Match Rate < 90%일 때 자동 개선 루프를 실행하여 갭을 줄이는 전문 매뉴얼

## 언제 사용
- QG4에서 Match Rate < 90%로 Act 단계 진입 시
- "갭 줄여줘", "자동 개선해줘", "Match Rate 올려줘"

## 사전 준비
1. `output/[시즌]/check/gap-report.md` → 갭 분석 결과 (필수 선행)
2. `output/[시즌]/check/sales-analysis.md` → 매출 분석
3. 각 갭의 원인 유형과 심각도 확인

## 자동 개선 루프

### 루프 규칙
- **최대 반복**: 5회
- **종료 조건**: Match Rate ≥ 90% 또는 최대 반복 도달
- **매 회차**: 원인 분석 → 액션 실행 → 재검증

### Step 1: 갭 우선순위 정렬
갭 리포트에서 심각도 기준 정렬:
1. 🔴 Critical (Gap > 20%) → 즉시 대응
2. 🟡 Warning (Gap 10~20%) → 개선 계획
3. 🟢 Minor (Gap < 10%) → 모니터링

### Step 2: 원인별 액션 매핑
| 원인 유형 | 대응 전략 | 담당 에이전시 |
|-----------|-----------|--------------|
| 예측 오류 | 예측 모델/기준 수정 | 전략기획실 |
| 실행 부족 | 실행 프로세스 보완 | 해당 에이전시 |
| 외부 변수 | 리스크 대응 매뉴얼 | 전략기획실 |
| 구조 문제 | 시스템/프로세스 개편 | QC 본부 |

### Step 3: 개선 액션 실행
각 액션별:
1. **현재 상태**: 갭 리포트의 해당 항목
2. **목표 상태**: 개선 후 기대치
3. **구체적 액션**: 무엇을 어떻게 변경
4. **담당**: 어떤 에이전시/역할
5. **검증 방법**: 어떻게 확인할 것인가

### Step 4: 재검증 (갭 디텍터 재실행)
- 개선 적용 후 Match Rate 재산출
- 변화량 기록: 이전 Match Rate → 현재 Match Rate
- 남은 갭 확인

### Step 5: 반복 또는 종료 판단
```
IF Match Rate ≥ 90% → 종료 → 완료 보고서 작성
IF 반복 횟수 ≥ 5 → 강제 종료 → 잔여 갭 기록
ELSE → Step 1로 복귀
```

## 완료 조건
- [ ] Match Rate ≥ 90% 달성 또는 최대 반복 도달

- [ ] 잔여 갭 기록 (해당 시)
- [ ] 완료 보고서 트리거 (리포트 제너레이터)

## 체크리스트

- [ ] 액션이 구체적이고 실행 가능한가?
- [ ] 재검증이 동일 기준으로 수행되었는가?
- [ ] 잔여 갭이 다음 시즌 Plan에 반영될 수 있는가?
