# 스킬: 테크팩 & 생산 관리 (Techpack & Production)

> 프로덕트 랩이 상품화를 진행할 때 따르는 실행 매뉴얼

## 언제 사용
- "테크팩 만들어줘", "원가 계산해줘", "QC 체크리스트 만들어줘"
- PDCA Do 단계에서 상품화 작업 시

## 테크팩 포맷

```markdown
# [아이템명] 테크팩

## 1. 기본 정보
| 항목 | 내용 |
|------|------|
| 스타일코드 | |
| 시즌 | |
| 카테고리 | |
| 라인 | |
| 소비자가 | |

## 2. 도식화 (Flat Sketch)
### 앞면
(도식화 텍스트 설명 — 또는 이미지 경로)
### 뒷면
(도식화 텍스트 설명)

## 3. BOM (Bill of Materials)
### 원단
| # | 소재명 | 조성 | 중량 | 색상 | 소요량 | 비고 |

### 부자재
| # | 부자재명 | 규격 | 색상 | 수량 | 비고 |

## 4. 사이즈 스펙 (단위: cm)
| 부위 | S | M | L | XL | 허용오차 |

## 5. 봉제 공정
| 공정 | 상세 | 주의사항 |

## 6. 그래픽/프린트 사양
| 위치 | 기법 | 사이즈 | 컬러 수 | 파일명 |

## 7. 포장 사양
| 항목 | 규격 | 비고 |

## 8. QC 체크 포인트
| # | 체크 항목 | 기준 | 불량 유형 |
```

## QR(Quick Response) 프로세스

호조상품 리오더 및 SPOT 트렌드 상품의 단기 생산:

```
[호조 시그널 감지] (매출 데이터/SNS 반응)
   │
   ▼
[기존 테크팩 확인] → 수정 필요 여부 판단
   │
   ├── 리오더 (동일 상품): 물량만 추가 발주
   │
   └── SPOT (신규/변형):
       ├── 간이 디자인 스펙 작성 (48시간 이내)
       ├── 기존 소재/부자재 활용 우선
       └── 단축 생산 일정 편성
```
