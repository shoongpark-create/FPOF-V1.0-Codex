# 테크팩 작성

> 프로덕션 매니저가 디자인을 생산 가능한 기술 문서로 변환하는 전문 매뉴얼

## 언제 사용
- "테크팩 만들어줘", "생산 문서 작성해줘"
- PDCA Do 단계에서 디자인 확정 후

## 사전 준비
1. `output/[시즌]/design/[아이템]-design-spec.md` → 디자인 상세 (필수 선행)
2. `output/[시즌]/design/[아이템]-costing.md` → 원가 검증 결과
3. `presets/wacky-willy/categories.json` → 카테고리별 사양 기준

## 테크팩 8대 섹션

### 1. 기본 정보
| 항목 | 내용 |
|------|------|
| 스타일코드 | |
| 시즌 | |
| 카테고리 | |
| 소비자가 / 원가 / 원가율 | |
| 생산 공장 / 납기일 | |
| 버전 | V1 |

### 2. 도식화 (Flat Sketch)
- 앞면: 모든 스티치라인, 봉제선 표시
- 뒷면: 앞면과 다른 디테일 명시
- 디테일 컷: 포켓 내부, 지퍼 구조, 라벨 위치
- 치수 기입점 표시

### 3. BOM (Bill of Materials)
#### 원단
| # | 소재명 | 조성 | 중량(g/m²) | 폭(inch) | 색상 | 소요량(yard) | 비고 |

#### 부자재
| # | 부자재명 | 규격 | 색상 | 수량 | 단가 | 비고 |

### 4. 사이즈 스펙 (단위: cm)
| 부위 | 측정 방법 | S | M | L | XL | 허용 오차 |

- 그레이딩 룰: 사이즈 간 증감폭
- 기본 사이즈: M (피팅 기준)

### 5. 봉제 공정
| 공정 | 상세 | 기계/기법 | 주의사항 |

- 특수 공정 표시 (워싱, 가먼트 다잉 등)
- 외주 공정 별도 표시

### 6. 그래픽/프린트 사양
| 위치 | 기법 | 사이즈(cm) | 컬러 수 | 팬톤 코드 | 파일명 |

- 위치 기준점 (어디에서 몇 cm)

### 7. 포장 사양
| 항목 | 규격 | 수량 | 비고 |

### 8. QC 체크 포인트
| # | 체크 항목 | 기준 | 불량 유형 | 허용 범위 |

- 불량 허용률: [X]% 이하
- AQL 기준

## 산출물 포맷

```markdown
# [아이템명] 테크팩

## 작성일: YYYY-MM-DD | 버전: V1
## 작성자: Production Manager

## 1~8 섹션 (위 구조대로)
```

- [ ] 사이즈 스펙 그레이딩 완료
- [ ] QC 체크포인트 설정
- [ ] 사용자/공장 승인

- [ ] 원가 계산서와 BOM이 정합성이 있는가?
- [ ] QC 기준이 브랜드 품질 수준에 맞는가?
