# 스킬: 품질 게이트 (Quality Gate)

> QC 본부가 단계 전환 시 품질 검증을 수행하는 매뉴얼

## 언제 사용
- PDCA 단계 전환 시 자동
- "검수해줘", "품질 체크해줘", "갭 분석해줘", "보고서 만들어줘"

---

## QG1: Plan → Design

### 검증 항목
| # | 체크 항목 | 기준 | 확인 방법 |
|---|----------|------|----------|
| 1 | 트렌드 브리프 존재 | 파일 존재 + 필수 섹션 완비 | output/[시즌]/plan/trend-brief.md |
| 2 | 브랜드 전략 브리프 존재 | 시즌 테마, 타겟 업데이트, 경영목표 연결 | output/[시즌]/plan/brand-strategy.md |
| 3 | 시즌 컨셉 노트 존재 | 카테고리 믹스, 가격 전략, 챔피언 상품 | output/[시즌]/plan/season-concept.md |
| 4 | 라인시트 존재 | SKU 리스트, OTB, 사이즈/컬러 | output/[시즌]/plan/line-sheet.md |
| 5 | 경영목표와 정합성 | 5대 목표 중 최소 2개와 명시적 연결 | 내용 확인 |
| 6 | 사용자 승인 | 산출물 검토 완료 | 승인 기록 |

### 판정
- **PASS**: 6/6 충족 → Design 진입
- **CONDITIONAL**: 4~5/6 충족 → 미비 항목 보완 후 재검증 또는 강제 통과
- **FAIL**: 3개 이하 → 보완 필수

## QG2: Design → Do

### 검증 항목
| # | 체크 항목 | 기준 |
|---|----------|------|
| 1 | 무드보드 존재 | 시즌 비주얼 방향 정의 |
| 2 | 디자인 스펙 존재 | 핵심 아이템 최소 1건 이상 |
| 3 | 원가 검증 완료 | 모든 아이템 원가 타겟 이내 (또는 VE 적용) |
| 4 | 디자인-기획 정합성 | 라인시트의 아이템과 디자인 스펙 매칭 |
| 5 | 비주얼 에셋 존재 | 최소 플랫 이미지 확보 |
| 6 | 사용자 승인 | 디자인 검토 완료 |

---

## QG3: Do → Check

### 검증 항목
| # | 체크 항목 | 기준 |
|---|----------|------|
| 1 | 테크팩 완성 | 런칭 아이템 전체 |
| 2 | 캠페인 브리프 존재 | IMC 전략 수립 완료 |
| 3 | 콘텐츠 기획서 존재 | 화보/영상 기획 완료 |
| 4 | 카피 데크 완성 | PDP + SNS 카피 |
| 5 | 런칭 시퀀스 확정 | 타임라인 + 담당자 배정 |
| 6 | 사용자 승인 | 런칭 준비 완료 확인 |

---

## QG4: Check → Next Cycle

### 검증 항목
| # | 체크 항목 | 기준 |
|---|----------|------|
| 1 | 매출 분석 완료 | 채널별/카테고리별 분석 |
| 2 | KPI 리뷰 완료 | 5대 경영목표 KPI 달성률 |
| 3 | 갭 분석 완료 | Match Rate 산출 |
| 4 | 인사이트 아카이빙 | 최소 1건 인사이트 카드 작성 |

### 판정
- **Match Rate >= 90%**: COMPLETE → 리포트 생성 → 다음 시즌 Plan
- **Match Rate < 90%**: Act 단계 → 자동 개선 (최대 5회) → 재검증

---

## Quality Gate 실행 리포트 포맷
```markdown
# Quality Gate [QG번호] 실행 결과

- 실행 일시:
- 단계: [현재] → [다음]
- 판정: [PASS / CONDITIONAL / FAIL]

## 체크 결과
| # | 항목 | 결과 | 비고 |

## 미비 항목 (해당 시)
| # | 항목 | 현재 상태 | 필요 액션 |

## 권고
- [다음 단계 진입 / 보완 후 재검증 / 강제 통과 요청]
```
