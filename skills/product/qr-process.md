# QR 프로세스 (Quick Response)

> 프로덕션 매니저가 호조상품 리오더와 SPOT 트렌드 상품을 단기 생산하는 전문 매뉴얼

## 언제 사용
- "이거 리오더 해야 해", "SPOT으로 빠르게 만들자"
- 인시즌 중 판매 데이터/트렌드 시그널 감지 시

## QR 유형 판단

| 구분 | 리오더 | SPOT |
|------|--------|------|
| 대상 | 기존 판매 호조 상품 | 트렌드 반응형 신규 상품 |
| 변경 | 물량만 추가 | 신규 디자인 (변형 가능) |
| 리드타임 | 2~3주 | 4~6주 |
| 테크팩 | 기존 그대로 | 간이 테크팩 신규 |
| 리스크 | 낮음 | 중간 |

## 리오더 프로세스
1. 판매율 확인 → 리오더 기준 충족? (예: 판매율 70% 이상)
2. 잔여 원단/부자재 확인 → 즉시 생산 가능?
3. 추가 발주 물량 산정 → 잔여 시즌 판매 예측 기반
4. 생산 공장 가용성 확인 → 납기일 확정
5. 발주 실행 → 입고 추적

## SPOT 프로세스
1. 트렌드 시그널 확인 → 어떤 트렌드에 반응?
2. 기존 소재/부자재 활용 가능성 → 리드타임 단축 핵심
3. 간이 디자인 스펙 작성 → 48시간 이내
4. 간이 원가 산출 → 기존 유사 아이템 기준
5. 간이 테크팩 작성 → 핵심 사양만
6. 단축 생산 일정 → 크리티컬 패스만

## 리스크 관리
- 재고 리스크: 시즌 잔여 기간 대비 물량 적정성
- 품질 리스크: 단축 공정에 따른 품질 저하 방지
- 타이밍 리스크: 트렌드 수명 vs 생산 리드타임

## 산출물 포맷

```markdown
# [상품명] QR 의뢰서

## 작성일: YYYY-MM-DD
## 유형: 리오더 / SPOT
## 긴급도: 높음 / 보통

## 근거
- 판매 데이터:
- 트렌드 시그널: (해당 시)

## 상품 정보
| 항목 | 내용 |
| 스타일코드 | (기존/신규) |
| 추가 물량 | |
| 목표 입고일 | |

## 생산 계획
| 단계 | 일정 | 비고 |

## 리스크 체크
- [ ] 시즌 잔여 기간 대비 물량 적정?
- [ ] 기존 품질 수준 유지 가능?
- [ ] 원부자재 확보 확인?
```

## 완료 조건
- [ ] QR 유형 판단 완료
- [ ] 물량/일정 확정
- [ ] 리스크 체크 완료
- [ ] 사용자 승인 → 발주 실행

## 체크리스트
- [ ] 판매 데이터 근거가 충분한가?
- [ ] 시즌 잔여 기간 대비 물량이 적정한가?
- [ ] 기존 소재/부자재 활용으로 리드타임을 단축했는가?
- [ ] 경영목표 #3(QR 비중) KPI에 기여하는가?
