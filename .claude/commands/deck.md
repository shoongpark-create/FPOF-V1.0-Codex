# FPOF 프레젠테이션 생성

패션 실무에 필요한 프레젠테이션 덱을 생성합니다. PPTX 스킬을 활용합니다.

## 사용법
`/deck [유형]` — 유형을 생략하면 현재 PDCA 단계에 맞는 덱을 추천합니다.

## 덱 유형

### Plan 단계
- `trend` — 트렌드 리서치 데크 (글로벌/로컬 트렌드, 키 비주얼, 시사점)
- `season` — 시즌 프레젠테이션 (테마, 컨셉, 컬러, 무드, 전략)
- `md-plan` — MD 전략 데크 (카테고리 믹스, 챔피언 상품, 가격 전략)

### Design 단계
- `moodboard` — 무드보드 데크 (비주얼 톤, 컬러 팔레트, 레퍼런스)
- `lookbook` — 룩북 프레젠테이션 (디자인 스케치, 스타일링 가이드)

### Do 단계
- `buyer` — 바이어 프레젠테이션 (라인시트, 사이즈런, 가격표)
- `campaign` — 캠페인 데크 (IMC 전략, 채널 플랜, 타임라인)

### Check 단계
- `report` — 시즌 리포트 데크 (매출 분석, KPI 대시보드, 인사이트)

## 절차

1. `.fpof-state.json`에서 현재 시즌과 단계 확인
2. `output/[시즌]/[프로젝트]/`에서 관련 산출물 읽기
3. `presets/wacky-willy/visual-identity.json`에서 브랜드 컬러/폰트 참조
4. PPTX 스킬을 사용하여 프레젠테이션 생성:
   - 브랜드 컬러 적용 (Primary: #FF6B35, Secondary: #2EC4B6 등 visual-identity.json 기반)
   - 와키윌리 톤앤매너 반영
   - 슬라이드 구성: 표지 → 목차 → 본문 → 요약 → 부록
5. `output/[시즌]/[프로젝트]/` 에 PPTX 파일 저장

## 슬라이드 디자인 원칙
- 브랜드 컬러 팔레트 필수 적용
- 텍스트는 간결하게, 비주얼 중심으로
- IP 캐릭터(키키) 활용 가능 시 포함
- 한 슬라이드 = 하나의 메시지

$ARGUMENTS 에 유형이 지정되면 해당 덱을, 없으면 추천합니다.
