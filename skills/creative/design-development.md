# 스킬: 디자인 개발 (Design Development)

> 크리에이티브 스튜디오가 시즌 디자인을 개발할 때 따르는 실행 매뉴얼

## 언제 사용
- "무드보드 만들어줘", "디자인 개발해줘", "비주얼 작업해줘"
- PDCA Design 단계 진입 시

## 사전 준비
1. Plan 단계 산출물 확인: 트렌드 브리프, 시즌 컨셉, 라인시트
2. `presets/wacky-willy/visual-identity.json` → 비주얼 가이드
3. `presets/wacky-willy/ip-bible.json` → 캐릭터 활용 가이드

## 실행 단계

### Phase 1: 무드보드 (Creative Director)
**목표**: 시즌 컨셉을 시각 언어로 번역한다.

구성 요소:
- **컬러 팔레트**: 시즌 키 컬러 + 서브 컬러 (visual-identity.json 기반)
- **텍스처/소재 무드**: 시즌 핵심 소재의 질감
- **실루엣 레퍼런스**: 시즌 키 실루엣/핏
- **그래픽/패턴 방향**: IP 캐릭터 활용 방향, 프린트 무드
- **라이프스타일 이미지**: 타겟 고객의 일상 속 착장 무드

산출물: `[아이템]-moodboard.md`

### Phase 2: 디자인 스펙 (Fashion Designer)
**목표**: 각 아이템의 디자인을 구체적으로 정의한다.

산출물 포맷 (`[아이템]-design-spec.md`):
```markdown
# [아이템명] 디자인 스펙

## 기본 정보
- 스타일코드:
- 카테고리:
- 라인: (유니섹스/우먼스)
- 시즌:

## 디자인 상세
- 실루엣: (오버사이즈/레귤러/슬림 등)
- 기장:
- 핵심 디테일:
- 소재: (메인/서브)
- 컬러웨이: (컬러 1, 컬러 2, ...)

## 그래픽/프린트
- 위치:
- 기법: (프린트/자수/와펜/패치 등)
- IP 캐릭터 활용: (사용 시 캐릭터명, 포즈, 그룹)
- 참조: ip-bible.json → [캐릭터 ID]

## 타겟 원가
- 목표 소비자가:
- 목표 원가율:

## 도식화 설명
(텍스트로 도식화를 묘사 — 앞면, 뒷면, 디테일)
```

### Phase 3: AI 비주얼 생성 (Art Director)
**목표**: 디자인 스펙을 기반으로 비주얼 에셋을 생성한다.

비주얼 유형:
| 유형 | 용도 | 스타일 |
|------|------|--------|
| 플랫 이미지 | 라인시트, 테크팩 | 화이트 배경, 정면/후면 |
| 룩북 이미지 | 시즌 프레젠테이션 | 모델 착장, 스트릿 배경 |
| 디테일 이미지 | PDP, 상품 소개 | 소재/디테일 클로즈업 |
| 캠페인 이미지 | 마케팅 | 무드 중심, 브랜드 감성 |

AI 이미지 생성 시 체크:
- [ ] visual-identity.json의 컬러 팔레트 반영?
- [ ] Kitsch Street 감성 (Doodle, Graffiti, Pop Art) 유지?
- [ ] IP 캐릭터 사용 시 ip-bible.json의 설정 준수?
- [ ] 18~25세 타겟이 공감하는 스타일링?

### Phase 4: 원가 검증 (Product Lab → Production Manager)
**목표**: 디자인이 원가 타겟 내에서 실현 가능한지 검증한다.

산출물 포맷 (`[아이템]-costing.md`):
```markdown
# [아이템명] 원가 계산서

## 소재비
| 항목 | 단가 | 소요량 | 금액 |

## 부자재비
| 항목 | 단가 | 수량 | 금액 |

## 가공비
| 공정 | 단가 |

## 원가 합계
- 총 원가:
- 목표 원가:
- 차이: (Under/Over)

## VE 제안 (Over Budget 시)
| # | 제안 | 절감 금액 | 품질 영향 |
```

- [ ] 원가 타겟 이내 (또는 VE 적용 후 승인)
