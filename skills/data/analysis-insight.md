- PDCA Check 단계 진입 시

1. `presets/wacky-willy/channels.json` → 채널별 목표치
2. `presets/wacky-willy/brand.config.json` → 경영목표 KPI 기준

## Part 2: 인사이트 도출 & 아카이빙 (Insight Architect)

### 사례 분석 프레임워크

**성공 사례 분석 시:**
```
1. 무엇이 성공했는가? (What)
   → 구체적 수치로 정의

2. 왜 성공했는가? (Why)
   → 외부 요인: 트렌드 타이밍, 시장 상황, 경쟁사 부재
   → 내부 요인: 상품력, 가격, 마케팅, 타이밍
   → 우연 vs 전략: 의도한 결과인가, 행운인가?

3. 반복 가능한가? (Reproducible?)
   → 어떤 조건에서 재현 가능한지
   → 필요한 전제 조건은 무엇인지

4. 원리화 (Principle)
   → "~한 조건에서 ~하면 ~한 결과를 기대할 수 있다"
```

**실패 사례 분석 시:**
```
1. 무엇이 실패했는가? (What)
   → 목표 대비 얼마나 미달?

2. 왜 실패했는가? (Why)
   → 예측 실패: 트렌드/수요 오판
   → 실행 실패: 타이밍, 품질, 마케팅 부족
   → 구조 실패: 프로세스/시스템 문제

3. 예방 가능한가? (Preventable?)
   → 어떤 시그널을 미리 잡았으면 피할 수 있었는지
   → 어떤 의사결정을 다르게 했어야 하는지

4. 교훈 (Lesson)
   → "~한 상황에서는 ~하지 말아야 한다" 또는
   → "~한 시그널이 보이면 ~을 점검해야 한다"
```

### 인사이트 카드 포맷
```markdown
## [인사이트 제목]
- **발견 일자**: YYYY-MM-DD
- **출처**: [시즌/상품/캠페인명]
- **카테고리**: [상품기획/마케팅/유통/글로벌/QR]
- **유형**: [성공원리/실패교훈/패턴발견/가설]
- **원리**: [한 문장으로]
- **근거 데이터**: [구체적 수치]
- **적용 조건**: [어떤 상황에서 유효]
- **한계/예외**: [적용되지 않는 경우]
- **관련 인사이트**: [기존 인사이트와의 연결]
- **다음 액션**: [어떻게 활용할 수 있는지]
```

### 플레이북 구조
시즌이 반복되면서 인사이트가 축적되면, 주제별 플레이북으로 통합:

```markdown
# [주제] 플레이북
> 최종 업데이트: YYYY-MM-DD | 기반 사례: N건

## 핵심 원리 (3~5개)
1.
2.
3.

## 체크리스트
- [ ] ...

## 사례 모음
### 성공 사례
### 실패 사례

## 주의사항
```

### 지식 색인 (`knowledge/index.md`)
```markdown
# 와키윌리 지식 아카이브

## 플레이북
| 주제 | 파일 | 최종 업데이트 | 기반 사례 수 |

## 시즌별 인사이트
| 시즌 | 파일 | 핵심 학습 수 |

## 사례 분석
| 사례명 | 유형 | 시즌 | 카테고리 | 파일 |

## 키워드 색인
| 키워드 | 관련 문서 |
```

## knowledge/ 디렉토리 관리

데이터 인텔리전스의 인사이트 아키텍트가 관리하는 핵심 자산:

```
knowledge/
├── index.md                        # 전체 지식 색인
├── playbooks/                      # 재생산 가능한 플레이북
│   └── (시즌이 쌓이면서 추가)
├── case-studies/                   # 개별 사례 분석
│   └── (시즌이 쌓이면서 추가)
└── insights/                       # 시즌별 인사이트 요약
    └── (시즌이 쌓이면서 추가)
```
